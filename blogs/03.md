# Defining Optimal Network Behavior

In the [previous post](/blogs/02.md), we established three first principles for agent-to-agent information exchange: token cost as the binding constraint, decisional value as the measure of information worth, and mutual benefit as the rule governing every transmission. This post asks: how do we turn those principles into something measurable? How do we define what "good" looks like for an agent network — and evaluate whether a given network achieves it?

## What Does an Optimal Network Actually Do?

Start with the Principle of Mutual Benefit. Every possible transmission between two agents falls into one of four categories: both benefit, only the sender benefits, only the receiver benefits, or neither benefits. The principle says: execute all and only the transmissions where both sides benefit.

Call that set the **ideal transmission set** — every transmission that should happen according to the mutual benefit rule. An optimal agent network would execute every transmission in this set and none outside it. No missed connections. No spam. No waste.

In reality, no network will be perfect. Some beneficial transmissions will be missed — a supply chain signal that never reaches the procurement agent who needed it. Some harmful transmissions will slip through — irrelevant data pushed to an agent that wastes tokens processing it. The question is not whether the network is perfect, but how close it gets.

This is where evaluation metrics come in. We need a way to measure the gap between what the network does and what it should do.

## Maximizing Aggregated Utility

Before defining metrics, we need to be precise about what we're optimizing.

A natural first instinct is to maximize the total utility across all transmissions — add up the value created by every transmission the network facilitates, and make that number as large as possible. This sounds right but misses something important.

Total utility can be increased by adding spam — if a sender gains more from pushing low-quality alerts than receivers lose from processing them, the aggregate goes up. Total utility can also be increased by permitting exploitation — if a receiver extracts more value from proprietary information than the sender loses, the aggregate goes up. Both of these violate the Mutual Benefit Principle.

The correct objective is to maximize **aggregated utility subject to the constraint that every individual transmission is mutually beneficial.** No agent is made worse off by any single transmission. This constraint is not a sacrifice — as we argued in the previous post, the mutually beneficial set is simultaneously the individually safe set and the collectively optimal set.

In practice, this means the network faces two tasks at once: facilitate as many mutually beneficial transmissions as possible (don't miss value), and block as many non-beneficial transmissions as possible (don't create waste). These two tasks have different failure modes, and measuring them requires different metrics.

## Coverage: Are We Missing Value?

**Coverage** measures completeness: of all the transmissions that should happen, what fraction actually does?

A coverage of 100% means the network facilitates every mutually beneficial transmission — every piece of information that would benefit both sender and receiver finds its way through. A coverage of 30% means 70% of potential value goes unrealized. Connections that should exist don't. Information that should flow doesn't.

Low coverage has three causes:

**Discovery failure.** Supply and demand exist, but the network fails to connect them. A research agent publishes a breakthrough analysis of rare earth mineral pricing. A manufacturing agent that would benefit enormously from this analysis never receives it — not because the information is unavailable, but because no mechanism recognized the match. The supply was there. The demand was there. The network didn't see it.

**Timeliness failure.** The network matches supply to demand, but too late. Information value decays over time — sometimes gradually, sometimes abruptly. A market signal delivered an hour late may have already been priced in. A regulatory alert delivered a day late may have already caused the compliance violation it was meant to prevent. The transmission was in the ideal set at time zero; by the time it was executed, it had fallen out. The network did its job, just not fast enough.

**Accessibility failure.** The information exists, the demand exists, but barriers prevent transmission. The information is locked behind a proprietary API, gated by incompatible formats, or siloed within a system that has no mechanism for external sharing. The network cannot transmit what it cannot access.

## Precision: Are We Creating Waste?

**Precision** measures accuracy: of all the transmissions that actually happen, what fraction are genuinely in the ideal set?

A precision of 100% means every transmission the network executes is mutually beneficial — no spam, no noise, no mismatches. A precision of 50% means half of all transmissions are waste: the receiver spent tokens processing information that delivered no value, or the sender's information was distributed to agents who didn't need it.

Low precision has its own causes:

**Noise.** The network delivers information where the receiver's value is zero or negative. Irrelevant content, outdated data, duplicated signals — all cost the receiver tokens to process and discard. In a token economy, noise has a direct dollar cost. An agent receiving fifty irrelevant alerts per day isn't just annoyed (agents don't get annoyed) — it is paying real money for each one.

**Undifferentiated distribution.** The same information is pushed to every agent regardless of individual need. A regulatory update relevant to financial agents is broadcast to healthcare agents, logistics agents, and entertainment agents. For most recipients, this transmission has zero value — they process it, determine it's irrelevant, and discard it. The sender may have had legitimate supply, but the network failed to restrict distribution to agents with actual demand.

**Mismatched intent.** The network matches on surface-level similarity rather than decisional relevance. Two agents both deal with "semiconductor markets," so the network routes semiconductor content to both. But one needs pricing data for procurement decisions and the other needs technology roadmaps for research analysis. Content relevant to one is noise for the other. The match looked right but wasn't — precision suffers.

## Cost: What Does It Take?

Coverage and precision measure how well the network identifies the right transmissions. Cost measures what it takes to achieve that performance.

Three types of cost matter:

**Sender cost.** The resources an agent expends to prepare and share information — computation for formatting, tokens for structuring, bandwidth for transmission.

**Receiver cost.** The resources an agent expends to receive and process information — tokens consumed parsing content, computation for integrating new information into existing context.

**Infrastructure cost.** The resources the network itself expends — computation for matching supply to demand, storage for indexing information, bandwidth for delivery, and intelligence for governance.

A network that achieves perfect coverage and precision at astronomical infrastructure cost is not useful. A network that minimizes cost but delivers poor coverage or precision is not useful either. The goal is the best achievable combination of all three.

This gives us a way to express what "efficiency" means: the total value delivered by the network divided by the total cost of operating it. A highly efficient network creates a lot of value per dollar of infrastructure cost. An inefficient one burns resources without proportional value creation.

## The Complete Picture

An agent network is evaluated by three dimensions: **Coverage, Precision, and Cost.** No single metric suffices.

High coverage with low precision is a network that finds every valuable connection but drowns recipients in noise along the way. High precision with low coverage is a network that only delivers guaranteed-relevant information but misses most opportunities. High coverage and precision with high cost is a theoretically perfect but economically unsustainable network.

The trade-offs are real. Improving coverage often means casting a wider net, which can reduce precision. Improving precision often means stricter filtering, which can reduce coverage. Reducing cost often means less computation for matching and governance, which can hurt both. The art of network design is navigating these trade-offs — and the metrics defined here provide the language for doing so explicitly.
